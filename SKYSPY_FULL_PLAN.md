# Sky Spy（空谍）完整开发方案

## 0. 当前定位

`Sky Spy（空谍）` 是一个 Java + libGDX 的 2D 实时动作 Roguelite。

当前版本不再是传统地牢探索，也不再以墙体迷宫为核心。游戏的第一体验应当是：

> 白雾中的随机浮岛，眼部投射，击退白色自己，把它们打下虚空，收集记忆碎片并继续向上。

核心约束：

- 地图是悬空浮岛和连接桥，不是封闭地牢。
- 地图、敌人、资源仍随机生成，保留 Roguelite 重开价值。
- 主角不绘制独立武器精灵，投射物从眼部方向发出。
- 所有敌人除猫以外，与主角共用同一套人形素材，只通过白色 tint、透明度、行为和特效区分。
- 白猫使用 Blind Jump 的 Laika 猫素材，是独立记忆锚点。
- 所有可见道具必须有交互或明确意义，不为了装饰硬塞元素。
- 入口是“记忆裂隙”，不使用旧入口语义或普通门语义。
- 玩家和敌人离开可行走浮岛会坠落死亡。

详细剧情见 [SKY_SPY_STORY_PLAN.md](SKY_SPY_STORY_PLAN.md)，实现符合度见 [SKY_SPY_COMPLIANCE_AUDIT.md](SKY_SPY_COMPLIANCE_AUDIT.md)。

## 1. 核心循环

单局循环：

1. 玩家醒在白雾浮岛。
2. 第一层先被白色自己围攻，首次脱困后触发顶层回响。
3. 探索随机浮岛，寻找记忆碎片、修补、回响和童年残留。
4. 用眼部投射、冲刺和走位处理敌人。
5. 利用击退把敌人打下平台，或被敌人逼到边缘而坠落。
6. 找到记忆裂隙，满足条件后进入下一层。
7. 死亡后泄露短句记忆，并保留部分局外醒念。
8. 第 5 层进入最终记忆验证与空自己对质。

玩家应当很快理解三件事：

- 边缘危险，坠落会死。
- 击退可以把敌人打下去。
- 白雾遮住未知区域，靠近后才知道那里有什么。

## 2. 叙事目标

游戏表层目标：

> 击落白色自己，穿过白雾浮岛，找到上层的叹息来源。

真实主题：

> 主角不是在杀怪物，而是在穿过被漂白的自我防御，逼自己重新看见完整的“我”。

层级结构：

| 层级 | 名称 | 玩法重点 | 叙事作用 |
|---|---|---|---|
| 1 | 白阶 | 基础白色自己、虚空边缘、首次回响 | 建立“这里敌视我” |
| 2 | 静默花园 | 远程敌人、讨好自己 | 引入“别出声” |
| 3 | 回声走廊 | 蓄力重击、白猫 Boss | 猫记忆开始清晰 |
| 4 | 镜面庭院 | 镜像自己、非敌对白猫 | 发现敌人越来越像自己 |
| 5 | 顶层房间 | 空自己、记忆验证 | 从杀穿转向整合 |

## 3. 系统命名

玩家可见文案必须使用 Sky Spy 语义。内部代码名可以暂时保留，避免无意义大范围重命名。

| 内部概念 | 玩家可见名称 | 当前含义 |
|---|---|---|
| floor | 意识层级 | 从底层记忆向顶层意识上升 |
| stairs / exit | 记忆裂隙 | 通往下一层的入口 |
| key | 记忆碎片 | 打开裂隙的条件 |
| gold | 回响 | 局内交换资源 |
| potion | 修补 | 恢复生命 |
| chest | 情绪匣 | 掉落资源或残留 |
| relic | 童年残留 | 形成 build 差异 |
| forge sparks | 醒念 | 死亡后保留的局外成长 |
| enemy | 白色自己 | 被漂白的自我片段 |
| boss | 守门自己 / 空自己 | 维护封锁的自我结构 |
| projectile | 眼部投射 | 当前主攻击表现 |

## 4. 技术基础

技术栈：

| 类别 | 技术 |
|---|---|
| 语言 | Java 17 |
| 框架 | libGDX 1.14.0 |
| 桌面后端 | LWJGL3 |
| 构建 | Gradle 9.1.0 wrapper |
| 字体 | FreeType + SmileySans |
| 渲染 | SpriteBatch / ShapeRenderer / Pixmap 生成贴图 |
| 地图 | 浮岛节点图 + Delaunay 风格候选边 + MST + loop 边 |
| 寻路 | A* 网格寻路 |

当前结构：

```text
core/src/main/java/com/kayro/dungeon/
  DungeonForgeGame.java        # 历史类名，负责 Screen 切换和偏好存档
  screen/                      # 菜单、游戏、暂停、死亡、结局
  world/                       # 世界、地图、生成器、房间/岛屿结构
  entity/                      # 玩家、敌人、投射物、物品、情绪匣
  system/                      # 输入、战斗、AI、寻路、刷怪、掉落、迷雾
  render/                      # 世界、HUD、小地图
  asset/                       # 资源加载、切片、程序贴图
  audio/                       # 音效池
  util/                        # 常量、方向、难度、数学工具
```

## 5. 资源方案

运行时素材以 `assets/` 为根目录加载。

当前原则：

- 优先使用 Blind Jump 现有素材。
- 不再使用旧地牢资源作为主要运行时素材。
- 不把门、急救箱等素材挪作无关对象。
- 没有合适素材时优先少放元素，而不是制造错误语义。
- `assets/title.png` 用作主菜单标题。
- `assets/fonts/SmileySans-Oblique-2.ttf` 用作全局中文字体。
- `assets/shaders/` 当前没有接入运行时代码，除非后续显式使用 `ShaderProgram`，否则只作为保留资源。

主要资源映射：

| 对象 | 当前策略 |
|---|---|
| 主角 | Blind Jump 人形素材 |
| 白色自己 | 主角素材 + 白色 hint/tint/透明度 |
| 白猫 | Blind Jump `gameObjects.png` 中 Laika 猫帧 |
| 浮岛地块 | Blind Jump tile/ground/edge 资源 |
| 记忆裂隙 | Blind Jump 圆形传送/能量入口资源 |
| 情绪匣 | 有明确交互的箱/容器资源 |
| 视觉遮罩 | 白雾、vignette、程序生成雾贴图 |
| 标题 | `assets/title.png` |

## 6. 地图生成

目标：

- 每次进入层级时地图不同。
- 所有关键区域可达。
- 避免不可通关小孤岛。
- 路宽至少 2 格。
- 场景有浮岛立体感，而不是平铺二维房间。

生成流程：

1. 生成若干浮岛节点，每个节点有中心、半径、房间权重和层级标签。
2. 为节点计算候选连接边，优先使用近邻边，效果接近 Delaunay 候选图。
3. 用 MST 保证所有关键节点连通。
4. 添加少量 loop 边，形成可绕路结构。
5. 以加权路径搜索连接节点，路径宽度至少 2 格。
6. 对边缘做二次标记，用正确的边缘瓦片绘制浮岛厚度。
7. 放置玩家、记忆碎片、情绪匣、敌人和记忆裂隙。
8. 运行连通性验证：出生点、裂隙、关键碎片必须可达。

验收：

- 不出现必须跳跃才能到达的关键区域。
- 不出现只剩 1 格宽的主通道。
- 不出现无法进入下一层的地图。
- 小地图能正确显示主连通区域和关键点。

## 7. 边缘与坠落

浮岛边缘是核心机制，不只是装饰。

规则：

- 玩家离开可行走区域会坠落死亡。
- 敌人离开可行走区域会死亡，并给玩家击落反馈。
- 子弹不再因为“墙”消失；当前场景没有传统墙体。
- 碰撞系统只关心可行走地块、实体碰撞和平台边界。
- A* 不应主动把敌人带向边缘死亡。

击退设计：

- 投射物命中敌人后施加击退。
- 敌人接近边缘时应尝试保守路径，而不是自己走下去。
- 玩家可以利用角度和站位把敌人打下虚空。

## 8. 输入与战斗

操作：

```text
WASD          移动
Shift         冲刺
鼠标左/右键   眼部投射
Q             使用修补
E             交互 / 打开情绪匣 / 进入裂隙
ESC           暂停
R             死亡后重试
Enter         菜单确认
```

眼部投射：

- 鼠标世界坐标决定方向。
- 投射物从角色眼部附近生成。
- 发射投射物不打断当前移动动画。
- 不单独播放“拔枪/举枪”动画。
- 面向屏幕时，绘制顺序需要保证投射物与角色遮挡关系自然。
- 其他方向按深度和朝向处理，不让子弹凭空消失。

伤害与反馈：

- 投射物造成伤害和击退。
- 白色外壳被击中时短暂裂开或降低白色遮罩。
- 击落敌人应给出明确文字、音效或资源回收反馈。
- 近距离接触和蓄力攻击仍应保留清晰预警。

## 9. 敌人与 AI

敌人原则：

- 除猫以外，敌人共享主角素材。
- 敌人差异来自行为、节奏、预警颜色、白色 hint 和层级透明度。
- 越往上，白色遮罩越弱，越接近“我”。

行为类型：

| 类型 | 行为 | 玩家解法 |
|---|---|---|
| 空白自己 | 基础追击 | 控制距离，击退下平台 |
| 逃避自己 | 远程攻击 | 拉近或绕视野 |
| 愤怒自己 | 蓄力重击/冲刺 | 看预警，侧闪 |
| 镜像自己 | 模仿/延迟追踪 | 改变节奏 |
| 守门自己 | 层级 Boss | 读阶段 |
| 白猫 | 特殊记忆锚点 | 第 4 层后避免误伤 |

A* 要求：

- 保留 A* 寻路。
- 代价函数要惩罚边缘格，降低敌人自己走下平台的概率。
- 目标点不可走或路径失败时回退到安全追踪，而不是直线送死。
- 敌人冲刺时也要保持朝向动画，不允许立正滑行。

## 10. 白雾视野

白雾是叙事和玩法核心。

视觉要求：

- 视野外是浓白雾，不是黑暗迷雾。
- 雾应按玩家距离逐渐变浓。
- 视觉视野可以比战斗信息计算范围略大，避免敌人突然贴脸出现。
- 不使用明显方块边界或多层矩形断层。
- 视野外敌人不显示本体、血条、轮廓、预警和文字信息。

当前建议：

- 用一层白雾颜色叠加 + 距离渐变控制浓度。
- 必要时使用 `vignetteMask.png` 或程序生成 radial fog。
- UI 永远在雾层上方绘制，不能被雾遮到看不清。

## 11. HUD 与界面

HUD 目标：

- 信息少而清楚。
- 不和场景、提示、暂停界面互相遮挡。
- 不给左上角状态区单独厚重背景。
- 操作提示只在游戏态显示，暂停/菜单/死亡界面隐藏。

HUD 布局：

| 区域 | 内容 |
|---|---|
| 左上 | 层级、生命、能量/回响条 |
| 右上 | 碎片、修补、回响、冲刺状态 |
| 右下 | 小地图 |
| 底部 | 操作提示，只在游戏态显示 |
| 角色附近 | 短提示，如 `E 使用`，必须有高对比描边或底衬 |

菜单要求：

- 开始、暂停、死亡、结局界面应统一继承基础遮罩/按钮/字体风格。
- 暂停、死亡、商店等界面尽量叠在已生成游戏画面上，而不是完全空白背景。
- 遮罩应保留场景可见度，不能把游戏界面完全洗掉。
- 主菜单可以使用白雾背景与大标题，不强制显示游戏场景。

## 12. 小地图

小地图功能：

- 显示已探索浮岛结构。
- 显示玩家位置。
- 显示记忆裂隙和关键碎片。
- 只显示符合视野/探索规则的信息。
- 不抢主视野，边框和点位颜色要统一。

关键点颜色建议：

| 对象 | 颜色 |
|---|---|
| 玩家 | 深蓝 |
| 记忆裂隙 | 青色或浅蓝 |
| 记忆碎片 | 黄色 |
| 敌人 | 仅在可见时显示 |

## 13. 道具与 Build

道具必须服务玩法或剧情。

可保留对象：

- 记忆碎片：打开裂隙。
- 修补：恢复生命。
- 回响：局内交换资源。
- 情绪匣：提供资源、童年残留或剧情短句。
- 童年残留：改变投射、冲刺、击退、生存方式。
- 白猫：剧情锚点，不作为普通奖励物。

应避免：

- 不可交互的随机摆件。
- 语义错误的素材挪用。
- 只增加视觉噪音的道具。

Build 方向：

| 残留 | 效果方向 |
|---|---|
| 裂白 | 打薄白色外壳，增加击落机会 |
| 回声瞳 | 强化眼部投射距离/命中反馈 |
| 残影步 | 冲刺后短时间强化投射 |
| 抱紧猫 | 受击减伤或坠落前短暂宽限 |
| 不出声 | 短时间降低敌人感知 |

## 14. 层级事件

每层应有一个简单可落地的叙事事件：

| 层级 | 必要事件 |
|---|---|
| 1 | 首次脱困后，靠近裂隙前触发顶层回响 |
| 2 | 讨好自己守门，掉落“冷地板/别出声”相关记忆 |
| 3 | 白猫半清晰 Boss，掉落“它趴在我旁边” |
| 4 | 白猫非敌对，玩家可以选择不攻击 |
| 5 | 空自己触发三段记忆验证 |

记忆验证：

- 第 2 层记忆回应讨好自己。
- 第 3 层记忆回应愤怒自己。
- 第 4 层记忆回应镜像自己。
- 三段完成后，空自己进入最终阶段。

## 15. 死亡与结局

死亡不是纯惩罚。

死亡结算应显示：

- 到达层级。
- 击落数量。
- 获得残留。
- 最强一击。
- 一句短记忆泄露。

普通结局：

- 击败空自己，但真结局条件不足。
- 看见治疗室灯光，意识苏醒但整合不完整。

真结局：

- 解锁关键死亡记忆。
- 第 4 层不攻击非敌对白猫。
- 最终战前选择等待。
- 结局台词落点是“我看见我了。猫也在。”

回顾模式：

- 真结局后的一次性后日谈。
- 攻击变成接触/安抚。
- 白色自己不再攻击。
- 文案提示为“不要再打了”或“不能再打了”一类短句，需与当前实现统一。

## 16. 声音

声音目标：

- 少而清楚。
- 不持续制造刺耳循环。
- 投射物飞行不应有过吵的连续噪音。
- 击中、击落、受伤、裂隙打开、记忆验证应有明确反馈。

音效接入要求：

- 只加载 libGDX 支持的音频格式。
- 24-bit WAV 需要转码或跳过，避免启动崩溃。
- 随机音效池要控制音量和触发频率。

## 17. 算法目标

当前项目应保留并强化以下算法点：

- 浮岛节点图生成。
- Delaunay 风格候选边。
- MST 连通保证。
- loop 边增加重玩变化。
- 加权通道搜索。
- A* 敌人寻路。
- 边缘风险代价。
- 视野距离渐变白雾。
- 难度随层级和表现微调。

不要为了“高级算法”牺牲可玩性。验收标准是地图可通、走位有意义、敌人不自杀、玩家能利用击退创造结果。
